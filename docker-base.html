Docker入门简明

邓亮泉 Docker

对Docker的简介网上有很多，我也就不说了，如果你不了解Docker是什么，可以先去网上查一查，Docker有三个最核心的概念，仓库(Repository)、镜像(Images)、容器(Container)，仓库可以看做是IPhone手机上的AppStore或者是Android手机上的应用市场，镜像可以看做是上架到应用商店的应用，容器这个概念比较不一样，可以把容器看做是镜像之上的一个空白层，你对镜像所产生的删除和添加都会在容器的这个空白层上，不会影响原有镜像如果你想一直应用变更你可以commit在这基础之上再派生一个镜像，Docker倡导Build once，Run anywhere。 Docker很像Git的管理机制，你可以对当前的变更进行提交也可以丢弃，也可以push上去把你的变更分享给他人使用，最好的学习方法还是实践，因为我使用的是Mac所以这里只说Mac的安装方法(Windows类似)，以后可能会写一篇关于Mac、Windows、Linux的安装吧，读者可以自行百度如何安装，很简单。 Mac/Windows用户可以从https://www.docker.com/toolbox 这个地方去下载Toolbox进行安装

Toolbox是一个安装器，目前支持Mac和Windows平台。使用它可以快速地在安装Docker工具集。本段翻译自Docker官方博客。Toolbox 可以安装你在开发中运行Docker时所需要的一切：Docker客户端、Compose（仅Mac需要）、Kitematic、Machine以及VirtualBox。Toolbox使用Machine和VirtualBox在虚拟机中创建了一个引擎来运行容器。在该虚拟机上，你可以使用Docker客户端、Compose以及Kitematic来运行容器。

我使用的Docker版本是v17.03.0-ce，安装完以后是想使用docker info查看docker信息测试是否安装成功，但是并不如意Error checking TLS connection: Host is not running出现了这个错误，提示也很明显就是说Docker的服务端没有启动(Docker是C/S架构)，我是这样做的

//因为待会我需要修改仓库地址，所以我可以先把默认的删除掉
> docker-machine rm default
//创建默认的并修改成阿里云的加速地址,加速地址可以自行去申请，如果你就像使用默认的地址可以把 --engine-registry-mirror 这个参数删掉
> docker-machine create --engine-registry-mirror=https://xxxxxx.aliyuncs.com -d virtualbox default
> docker-machine env default
> eval "$(docker-machine env default)"
> docker info

建议将eval "$(docker-machine env default)"写入.bash_profile之类的环境文件中去，这样就不用每次新开命令行窗口都要执行一遍这个了，重启电脑后Docker是不会自动启动的，可以执行docker-machine start default，或配置成开机启动。 环境搭建好了后开始进行实践，按照上图首先我们需要一个系统镜像(这里我是使用Ubuntu)可以去docker hub上去找 ，由于我是使用的aliyun的镜像仓库( 开发者平台 )我便直接在aliyun的镜像仓库找了

//首先需要将仓库的镜像拉取至本地
> docker pull learn/tutorial 
//查看本地的镜像
> docker images
//利用pull至本地的镜像run一个容器
> docker run learn/tutorial echo "Hello World!"
Hello World

实际上你在执行run的时候，如果镜像不存在本地，它会自己去仓库搜索这个镜像pull下来.该镜像是用的默认源，这里可以参照 Ubuntu镜像使用帮助修改源，修改完后记得执行sudo apt-get update

//通过命令查看运行过的容器
> docker ps -a 
//安装ping 
> docker run learn/tutorial  apt-get install -y ping
//可以查看到刚刚执行的docker容器，复制它的container id
> docker ps -l
//提交，在原有镜像的基础上增加了ping，生成新的镜像
> docker commimt 6a713461237b dlq/ping
//查看镜像
> docker images
//使用刚自行创建的镜像运行一个容器进行操作
> docker run dlq/ping ping www.dlq.wiki

当然你可以使用docker ps -a 查看你所有运行过的容器，想删除的可以通过docker rm 容器ID如果想删除镜像可以通过docker rmi 镜像名/ID

> docker run dlq/ping apt-get install -y wget
> docker ps -l
> docker commit 50f0371d625a dlq/ping-wget
> docker run dlq/ping-wget wget http://dlq.wiki/

> docker images
> docker run dlq/ping-wget apt-get  remove -y iputils-ping
> docker ps -l
> docker commit 22ce090470e9 dlq/wget

^ ^ 文章到此结束，你还可以使用docker push将你的镜像推送至仓库，具体操作可以参照Docker官网文档，以后可能还会在写一篇《Docker构建开发环境And线上》
